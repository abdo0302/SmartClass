Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

POST api/auth/register

Example request:
curl --request POST \
    "http://localhost/api/auth/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"sjuodr\",
    \"email\": \"elwyn27@example.net\",
    \"password\": \"5?f8j\\/h\",
    \"role\": \"professeur\"
}"

const url = new URL(
    "http://localhost/api/auth/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "sjuodr",
    "email": "elwyn27@example.net",
    "password": "5?f8j\/h",
    "role": "professeur"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST api/auth/register

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

name   string   

Must not be greater than 255 characters. Example: sjuodr

email   string   

Must be a valid email address. Must not be greater than 255 characters. Example: elwyn27@example.net

password   string   

Must be at least 8 characters. Example: 5?f8j/h

role   string   

Example: professeur

Must be one of:
  • eleve
  • professeur
